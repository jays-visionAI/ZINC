# 🔒 네, 정확합니다. (Code vs Data 분리 원칙)

말씀하신 부분이 바로 **Hybrid Architecture**의 핵심 안전장치입니다.

### 1. 소스코드 (Engine) = 🧊 변경 불가능 (Immutable)
*   **저장소:** 서버(Cloud Functions) 및 코드베이스
*   **변경 방법:** 오직 **개발자가 직접 코드를 수정하고 배포(Deploy)** 해야만 바뀝니다.
*   **안전장치:** AI가 스스로 이 코드를 건드리는 것은 **물리적으로 불가능**합니다. (Read-only)

### 2. 프롬프트 & 설정 (Instruction) = 📝 버전 관리 (Versioned)
*   **저장소:** Agent Registry (DB)
*   **변경 방법:** Admin UI에서 수정하거나, Growth 시스템이 제안합니다.
*   **안전장치:** 변경되더라도 **항상 새로운 버전(v1.1, v1.2)으로 저장**되므로, 언제든 과거 버전(v1.0)으로 즉시 롤백할 수 있습니다.

**결론:**
유저님이 "승인"하거나 "배포"하지 않는 한, 에이전트의 핵심 로직이 **몰래 바뀌는 일은 절대 없습니다.** 안심하셔도 됩니다.
